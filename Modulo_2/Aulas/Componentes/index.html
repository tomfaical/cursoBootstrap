Estrutura e boas práticas de commits

Commit Msg Descrição
feat adicionar funcionalidade
fix corrigir bug
chore alteração que não impacta o funcionamento do código (ex. atualizar versões de libs)
refactor reestruturação de código sem mudar o comportamento (ex. melhorar lógica de autenticação)
docs alteração na documentação
test add um teste
style mudanças estéticas no código (como indentação e remoção de espaços)
perf melhorias de performance
